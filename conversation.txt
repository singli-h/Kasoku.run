Based on everything gathered, here are 3 architectural approaches for the AI-assisted coach planning workflow. Each addresses: group-first planning, subgroup differentiation (100m/200m/400m), individual overrides, and context flowing across macro → meso → microcycle.

3 Architecture Approaches
Option A: Dedicated Context Records (Recommended)
New planning_contexts table with records at macro and meso levels. Each context holds: seasonal direction, phase objectives, event-group variants (JSONB array with per-event-group direction), constraints, and AI-extracted structured fields.

Separately, a microcycle_insights table captures what actually happened each week (volumes, readiness flags, athlete deviations). When AI generates the next microcycle, it receives the full chain: season context → phase context → last 2-3 weeks of insights → athlete profiles/notes.

Subgroup differentiation lives inside the context record as an event_group_variants array: [{ group: "sprint", direction: "..." }, { group: "mid-distance", direction: "..." }]. Athletes have an event_group tag. Individual overrides get their own athlete_plan_overrides table keyed by (session_plan_id, athlete_id).

Trade-off: More schema work upfront, but AI gets clean structured input, it's queryable, and it scales to multi-season memory.

Option B: Extend Existing Fields (Lightweight / Ship Fast)
Reuse the existing macrocycles.notes and mesocycles.metadata JSONB columns — both already exist and are already ignored by AI. Coach types direction into these fields. A thin extraction step converts free-text + conversation into structured fields stored back to JSONB.

Weekly insights stored in microcycles.metadata. No new tables. The AI routes get enhanced to pull from this chain before generating sessions.

Trade-off: Ships in days, not weeks. But notes and metadata fields were designed for other purposes and will get messy. Harder to query, harder to extract reliably. Good for a quick proof-of-concept but creates tech debt.

Option C: Persistent Planning Threads (Conversation-First)
Each macrocycle and mesocycle gets a persistent AI conversation thread stored in a planning_threads table. Coach "plans" by chatting — AI asks questions, extracts decisions, and saves them as structured summaries. No separate form to fill.

The thread summary is injected into microcycle generation. Works like a briefing: "Here's what the coach decided for this phase and why." Subgroups defined during the thread conversation and stored in the extracted summary.

Trade-off: Most natural UX for coaches who hate forms. But complex to implement reliably — AI summarization can miss things, conversation threads get long, harder to edit/correct decisions after the fact.

My Recommendation: Option A
Option A wins because:

AI needs structured, reliable input — free-text context (Option B/C) creates extraction failures at the worst time (when generating the actual plan)
The event_group_variants array elegantly handles 100m/200m/400m differentiation without multiplying tables
microcycle_insights is the missing link nobody has — this is the "previous microcycle insight" that feeds the next generation and becomes your multi-season memory moat
Option B is a shortcut that becomes a liability when a coach has 12 athletes across 3 event groups over 3 seasons
Option C's conversation-first UX is appealing — I'd blend it into A: the input mechanism for creating the context record is a hybrid AI conversation, but the storage is always structured (Option A's schema).

# Coach AI Planning Design

**Status**: In progress — brainstorming
**Date**: 2026-03-05
**Feature**: AI-assisted coach planning workflow

---

## Problem Statement

Coach MesoWizard currently has **zero AI**. Planning context (direction, phase objectives, coaching intent) is never fed into AI generation. The individual athlete path has AI via `QuickStartWizard` + init pipeline, but coaches managing groups have no equivalent.

Key gaps:
- No way to record season/phase direction at the macro/meso level
- AI generates microcycles without any coaching context
- No previous microcycle insights flowing into next week's generation
- No subgroup (event specialty) awareness in planning

---

## Goals

- Initial season blueprint creation with AI assistance
- Ongoing weekly microcycle refinement with AI
- Context persists and flows: macro → meso → micro
- Previous microcycle insights feed into next generation
- Group-first planning: one plan for a squad, with subgroup and individual differentiation

---

## What We Know So Far

### Input Method
Hybrid: AI conversation + free-text notes. Coach chats with AI to develop direction; output is saved as structured context. Free-text notes also accepted directly.

### Planning Hierarchy Confirmed

```
Coach
 └── Squad (GHS / PC / SiChuan — different groups/schools)
      └── Subgroup by event specialty (SS / MS / LS)
           └── Individual athlete (with overrides)
```

### Real-World Pattern (from CSV analysis)

A coach manages **multiple groups** (different schools/clubs) that often run **the same underlying program** with different training frequencies and schedules:

- GHS — 3x/week (Mon/Wed/Fri)
- PC — 2x/week (Tue/Thu)
- SiChuan — variable schedule

Within each group, subgroups (SS/MS/LS) are differentiated **inline within sessions** — not separate plans. E.g.: `LS: 15 x 200m M:36" F:39" | SS/MS: CSD @80%`. Gender targets also inline.

**Key insight**: It's ONE shared periodization philosophy adapted for different group schedules. Not N independent plans.

/**

```typescript
export async function saveMacroPlanningContextAction(

  } catch (e) {
    return { isSuccess: false, message: String(e) }
  }
}

export async function saveMesoPlanningContextAction(

): Promise<ActionState<void>> {

    const { userId } = await auth()
```

**Files:**

**Step 1: Write the action**

```typescript
'use server'

import { auth } from '@clerk/nextjs/server'
import { getDbUserId } from '@/lib/user-cache'
import supabase from '@/lib/supabase-server'
import type { ActionState } from '@/types/actions'

interface GeneratedMicrocycleContext {
  macroContext: string | null
  mesoContext: string | null
  recentInsights: string[]
  athleteEventGroups: string[]
  upcomingRaces: string[]
  schedule: string | null
}

/**
 * Fetch all context needed for AI to generate a microcycle for a group.
 * Returns a structured object ready to inject into AI system prompt.
 */
export async function getMicrocycleGenerationContextAction(
  microcycleId: number,
  athleteGroupId: number
): Promise<ActionState<GeneratedMicrocycleContext>> {
  try {
    const { userId } = await auth()
    if (!userId) return { isSuccess: false, message: 'Not authenticated' }
    const dbUserId = await getDbUserId(userId)

    // 1. Get microcycle + meso + macro chain
    const { data: micro, error: microError } = await supabase
      .from('microcycles')
      .select(`
        id, start_date, end_date, mesocycle_id,
        mesocycles!inner (
          id, metadata,
          macrocycles!inner (
            id, planning_context
          )
        )
      `)
      .eq('id', microcycleId)
      .single()

    if (microError || !micro) return { isSuccess: false, message: 'Microcycle not found' }

    const meso = Array.isArray(micro.mesocycles) ? micro.mesocycles[0] : micro.mesocycles
    const macro = Array.isArray(meso?.macrocycles) ? meso.macrocycles[0] : meso?.macrocycles

    // 2. Get last 3 weekly_insights for this group
    const { data: pastMicros } = await supabase
      .from('microcycles')
      .select('weekly_insights, start_date')
      .eq('athlete_group_id', athleteGroupId)
      .not('weekly_insights', 'is', null)
      .lt('start_date', micro.start_date ?? new Date().toISOString())
      .order('start_date', { ascending: false })
      .limit(3)

    // 3. Get athlete event_groups in this group
    const { data: athletes } = await supabase
      .from('athletes')
      .select('event_group')
      .eq('athlete_group_id', athleteGroupId)
      .not('event_group', 'is', null)

    // 4. Get upcoming races in this microcycle's date range
    const { data: races } = await supabase
      .from('races')
      .select('name, date, category')
      .gte('date', micro.start_date ?? '')
      .lte('date', micro.end_date ?? '')
      .order('date')

    const macroContext = macro?.planning_context
      ? JSON.stringify(macro.planning_context)
      : null

    const mesoMeta = meso?.metadata as Record<string, unknown> | null
    const mesoContext = mesoMeta?.planning_context
      ? String(mesoMeta.planning_context)
      : null

    const recentInsights = (pastMicros ?? []).map(m => {
      const insights = m.weekly_insights as Record<string, unknown> | null
      return insights?.summary ? String(insights.summary) : ''
    }).filter(Boolean)

    const athleteEventGroups = [
      ...new Set((athletes ?? []).map(a => a.event_group).filter(Boolean) as string[])
    ]

    const upcomingRaces = (races ?? []).map(r => `${r.name} on ${r.date}`)

    const scheduleNote = mesoMeta?.schedule_notes ? String(mesoMeta.schedule_notes) : null

    return {
      isSuccess: true,
      message: 'Context loaded',
      data: {
        macroContext,
        mesoContext,
        recentInsights,
        athleteEventGroups,
        upcomingRaces,
        schedule: scheduleNote,
      }
    }
  } catch (e) {
    return { isSuccess: false, message: String(e) }
  }
}
```

### Task 11: summarizeWeeklyInsightsAction

AI drafts weekly_insights from workout_logs after a microcycle ends.

**Files:**

**Step 1: Write the action**

```typescript
'use server'

import { auth } from '@clerk/nextjs/server'
import { getDbUserId } from '@/lib/user-cache'
import supabase from '@/lib/supabase-server'
import type { ActionState } from '@/types/actions'

interface WeeklyInsightsDraft {
  summary: string
  completionRate: number       // 0–1
  keyObservations: string[]
  suggestedAdjustments: string[]
  generatedAt: string
}

/**
 * Fetch workout_logs for a completed microcycle and build a raw summary
 * for use in AI prompt. Client calls the AI route, then calls saveWeeklyInsightsAction.
 */
export async function getMicrocycleWorkoutSummaryAction(
  microcycleId: number
): Promise<ActionState<{ microcycleName: string | null; workoutSummary: string }>> {
  try {
    const { userId } = await auth()
    if (!userId) return { isSuccess: false, message: 'Not authenticated' }
    await getDbUserId(userId)  // verify user exists

    const { data: micro } = await supabase
      .from('microcycles')
      .select('name, start_date, end_date')
      .eq('id', microcycleId)
      .single()

    // Get all workout_logs for sessions in this microcycle
    const { data: logs } = await supabase
      .from('workout_logs')
      .select(`
        status,
        effort_percentage,
        athlete_notes,
        session_plans!inner (
          name,
          microcycle_id
        )
      `)
      .eq('session_plans.microcycle_id', microcycleId)

    if (!logs || logs.length === 0) {
      return { isSuccess: false, message: 'No workout data for this microcycle' }
    }

    const completed = logs.filter(l => l.status === 'completed').length
    const total = logs.length
    const avgEffort = logs
      .filter(l => l.effort_percentage != null)
      .reduce((sum, l) => sum + (l.effort_percentage ?? 0), 0) / Math.max(1, logs.filter(l => l.effort_percentage != null).length)

    const athleteNotes = logs
      .filter(l => l.athlete_notes)
      .map(l => l.athlete_notes)
      .join('\n')

    const summary = [
      `Week: ${micro?.name ?? microcycleId}`,
      `Dates: ${micro?.start_date} to ${micro?.end_date}`,
      `Completion: ${completed}/${total} sessions`,
      `Avg effort: ${Math.round(avgEffort)}%`,
      athleteNotes ? `Athlete notes:\n${athleteNotes}` : '',
    ].filter(Boolean).join('\n')

    return { isSuccess: true, message: 'Summary ready', data: { microcycleName: micro?.name ?? null, workoutSummary: summary } }
  } catch (e) {
    return { isSuccess: false, message: String(e) }
  }
}

/**
 * Save confirmed weekly_insights JSONB to a microcycle.
 */
export async function saveWeeklyInsightsAction(
  microcycleId: number,
  insights: WeeklyInsightsDraft
): Promise<ActionState<void>> {
  try {
    const { userId } = await auth()
    if (!userId) return { isSuccess: false, message: 'Not authenticated' }
    const dbUserId = await getDbUserId(userId)

    const { error } = await supabase
      .from('microcycles')
      .update({ weekly_insights: insights })
      .eq('id', microcycleId)
      .eq('user_id', dbUserId)

    if (error) return { isSuccess: false, message: error.message }
    return { isSuccess: true, message: 'Weekly insights saved' }
  } catch (e) {
    return { isSuccess: false, message: String(e) }
  }
}
```

```typescript
  mesocycleId: z.coerce.number().int().positive().optional(),
  existingContext: z.string().max(5000).optional(),
  importText: z.string().max(20000).optional(),
})

const SYSTEM_PROMPT = `You are an expert sports science consultant helping a coach structure their training plan.

Your role is to have a natural conversation to understand:
1. The season goals and competition targets
2. The training philosophy (volume-first, intensity-first, event-specific focus)
3. The athlete group composition (event specialties, experience levels)
4. The training schedule (days per week per group, facilities available)
5. Any constraints (injuries, competitions, season start)

Be conversational and practical. Ask one focused question at a time.
When you have enough context, summarize what you've learned in a clear planning brief.

If the coach provides imported text (CSV, document), extract the relevant planning philosophy from it.
Focus on extracting: periodization structure, weekly training load, event-specific prescriptions, competition calendar.

Always end your response with a concise "Planning Summary:" section when you feel you have enough context.`

export async function POST(req: Request) {
  let userId: string | undefined
  try {
    const [authResult, body] = await Promise.all([auth(), req.json()])
    userId = authResult.userId
    if (!userId) return new Response('Unauthorized', { status: 401 })

    if (process.env.AI_ENABLED === 'false') {
      return new Response('AI features temporarily unavailable', { status: 503 })
    }

    const { allowed } = checkServerRateLimit(userId, 20, 60_000)
    if (!allowed) return new Response('Too many requests', { status: 429 })

    try { await getDbUserId(userId) }
    catch { return new Response('User not found', { status: 404 }) }

    let validated: z.infer<typeof RequestSchema>
    try { validated = RequestSchema.parse(body) }
    catch (e) {
      return new Response(JSON.stringify({ error: 'Invalid request' }), {
        status: 400, headers: { 'Content-Type': 'application/json' }
      })
    }

    const systemParts: string[] = [SYSTEM_PROMPT]
    if (validated.existingContext) {
      systemParts.push(`\nExisting planning context (coach has already shared this):\n${validated.existingContext}`)
    }
    if (validated.importText) {
      systemParts.push(`\nCoach imported this training document — extract planning philosophy:\n${validated.importText}`)
    }

    const result = streamText({
      model: openai('gpt-4o'),
      system: systemParts.join('\n\n'),
      messages: convertToModelMessages(validated.messages as UIMessage[]),
      experimental_transform: smoothStream(),
      maxSteps: 3,
    })

    return result.toDataStreamResponse()
  } catch (e) {
    console.error('[planning-context-chat] Error:', e)
    return new Response('Internal server error', { status: 500 })
  }
}
```

```tsx
'use client'

import { ErrorBoundary } from 'react-error-boundary'
import { AlertTriangle } from 'lucide-react'
import { TrainingPlanWorkspace } from '../workspace/TrainingPlanWorkspace'
import { SeasonContextPanel } from './SeasonContextPanel'
import type { TrainingPlan } from '../workspace/TrainingPlanWorkspace'

interface CoachPlanPageWithAIProps {
  initialPlan: TrainingPlan & {
    planning_context?: unknown | null
  }
  coachGroups?: Array<{ id: number; name: string }>
}

function CoachPlanFallback({ error, resetErrorBoundary }: { error: Error; resetErrorBoundary: () => void }) {
  return (
    <div className="bg-destructive/10 border-b border-destructive/20 px-4 py-3">
      <div className="flex items-center gap-3">
        <AlertTriangle className="h-5 w-5 text-destructive" />
        <div>
          <p className="text-sm font-medium text-destructive">Planning context unavailable</p>
          <p className="text-xs text-muted-foreground">You can still edit your training plan.</p>
        </div>
        <button
          onClick={resetErrorBoundary}
          className="ml-auto px-3 py-1.5 bg-primary text-primary-foreground rounded-md text-xs"
        >
          Retry
        </button>
      </div>
    </div>
  )
}

export function CoachPlanPageWithAI({ initialPlan, coachGroups = [] }: CoachPlanPageWithAIProps) {
  const planningContextText = (() => {
    const ctx = initialPlan.planning_context
    if (!ctx) return null
    if (typeof ctx === 'string') return ctx
    const obj = ctx as Record<string, unknown>
    return typeof obj.text === 'string' ? obj.text : JSON.stringify(ctx)
  })()

  return (
    <ErrorBoundary fallbackRender={({ error, resetErrorBoundary }) => (
      <CoachPlanFallback error={error} resetErrorBoundary={resetErrorBoundary} />
    )}>
      <div className="flex flex-col">
        <SeasonContextPanel
          macrocycleId={initialPlan.macrocycle.id}
          planningContext={planningContextText}
        />
        <TrainingPlanWorkspace
          initialPlan={initialPlan}
        />
      </div>
    </ErrorBoundary>
  )
}
```

**Mandatory:**
- After each task in subagent-driven development
- After completing major feature

**Optional but valuable:**
- When stuck (fresh perspective)
- Before refactoring (baseline check)
- After fixing complex bug

**1. Get git SHAs:**
```bash
BASE_SHA=$(git rev-parse HEAD~1)  # or origin/main
```

- Fix Critical issues immediately
- Fix Important issues before proceeding
- Note Minor issues for later
- Push back if reviewer is wrong (with reasoning)
