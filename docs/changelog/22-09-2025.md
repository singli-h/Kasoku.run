PRD-style overview (concise)
1) Problem / Goal

Provide a lean, scalable schema that supports:

A shared + custom exercise catalog retrievable by tags and vector search, with coach-wide visibility.

A unified memory system for coach/athlete/group notes (preferences, injuries, session summaries) usable by AI later.

Keep tables minimal; avoid premature fragmentation; preserve historical session data when exercises change.

2) What’s in scope (this change)

Add exercise embedding, FTS, ownership/visibility, archiving.

Add tag families (category) for precise filtering.

Create memories (polymorphic, typed) + updated_at trigger.

No changes to presets/sessions or RLS in this migration.

3) Non-goals (now)

No per-tag bias tables; AI will reason from memory.

No cross-coach/partial-group sharing flows.

No RLS policy changes (handled separately).

No SQL-level contraindication filtering (the agent will consider memory context).

4) Key design decisions

exercises.visibility enum = global | coach | group | user

global: app-provided, no owner; immutable re: historical sessions (see §7).

coach: owned by a coach (via owner_user_id), visible across all that coach’s groups.

user: private to a single user (e.g., athlete).

Search stack: scope by visibility/ownership → optional tag family filters → ANN over embedding → (small) prompt.

Memories: one table with separate foreign key columns (coach_id, athlete_id, group_id), memory_type (incl. session_summary), metadata (e.g., { "session_id": 123 }).
Exactly one FK must be populated per memory. Recency via updated_at; we can encode progression/time in metadata and future embeddings.

5) FE/BE feature requirements (MVP)

Exercise Catalog

Create/Edit Exercise:

Fields: name, description, video_url, type, unit, tags[], visibility (radio), group selector (if group), Archive toggle.

Ownership auto-set from current user.

Search/Picker for AI & UI:

Filters: visibility scope (default: include global + current coach + current user), tag families, exclude archived.

Sorting: vector similarity (server tool), fallback FTS on search_tsv.

Integrity: If an exercise is edited, historic sessions do not change (sessions store resolved exercise references + performed values).

Memories

Create memory for Coach/Athlete/Group:

memory_type: preference | philosophy | injury | profile | note | session_summary

content (short free text), metadata (key/values; include session_id when session summary).

Populate exactly one of: coach_id, athlete_id, or group_id.

Auto-update updated_at via trigger.

Memory list:

Filter by subject (coach_id, athlete_id, or group_id) + type; sorted by updated_at DESC.

(Future) semantic search across memory.embedding.

AI Tooling (backend)

Tool: search_exercises({ scope, tags, query_embedding, exclude_archived }) -> [exercise]

scope derived from current user & context (global + coach + group + user).

Returns top-K with id,name,description,tags,visibility.

Tool: list_memories({ coach_id?, athlete_id?, group_id?, memory_type? })

Used for pulling preferences/injuries/session summaries into prompts. Exactly one subject ID must be provided.

6) Data retention / historical integrity

Training sessions are immutable records. Editing an exercise (name/description/tags) does not rewrite past exercise_training_* rows.

Archiving an exercise only hides it from pickers; historical references remain valid.

7) Performance & indexing

ANN on exercises.embedding (ivfflat, lists=100; tune later).

GIN on search_tsv and metadata.

Composite (visibility, owner_user_id) for scoping.

8) Telemetry (nice-to-have soon)

Log tool query shapes (filters, counts, latency).

Track AI pick vs. user override to refine retrieval.

9) Risks / mitigations

Typos in tags.category → guarded by CHECK.

Enum rigidity → values stable; adding new ones is a simple migration.

Separate FK columns (coach_id, athlete_id, group_id) with proper constraints → data integrity enforced at database level.

Summary of feature expectations (concise)

Exercises

Support global (app), coach-wide, single-group, and user-private exercises.

Fast retrieval via scope → tags → vector similarity.

Archiving removes from pickers, keeps history intact.

Memories

One table for coach/athlete/group knowledge with separate FK columns.

Types include session_summary for brief post-session notes/performance pointers.

Recency via updated_at; optional time series in metadata.

Cascade deletes ensure data consistency when subjects are removed.

No schema change to presets/sessions now.
