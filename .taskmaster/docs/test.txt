Below is a complete “pre-work” package you can drop straight into Task Master (or adapt manually) before you start invoking web-eval-agent. It has two parts:
A. Requirements & Test-plan scope
B. Task-list (parent task + granular subtasks) ready for Task Master
Feel free to tweak titles or wording, but this gives full coverage of every UI/UX feature currently in the repo.
A. Requirements & Test-Plan Scope
1. Objective
Automate a thorough UX / UI smoke-test of the entire web app using web-eval-agent.
Surface any visual, functional, or flow breakages and log them with screenshots.
Coverage Matrix
Marketing pages
Home /
Feature sections, CTA buttons, footer / header links
Auth flows (/sign-in, /sign-up)
Form validation, error messaging, social login placeholders
Onboarding wizard (/onboarding)
Organization creation, role selection, progress indicators
Core signed-in shell (Sidebar + Header)
Collapse / expand sidebar, route navigation, dark-mode toggle
Dashboard (/dashboard)
KPI cards render, recent activity list, responsive layout
Athletes module (/athletes)
List, “Add athlete”, view detail, edit, delete
Workout Builder (/workout)
Create workout, add exercise, save, edit
Plans (/plans)
Create plan, assign athletes, pagination
Performance Analytics (/performance)
Charts render, filter by athlete / date
Library (/library)
Knowledge-base articles list + detail
Settings (/settings)
Profile tab, billing tab, theme switcher
Global error & 500 pages
Force 404 and 500 routes, verify copy & “Go home” link
Responsive checks
Desktop ≥1280 px, Tablet 768 px, Mobile 375 px
Evaluation Criteria (used in every test step)
Page reaches OK (200) or intended status
No console errors
Main components visible and interactive
Layout intact on desktop & mobile
Semantic focus order and aria-labels exist (accessibility spot-check)
Any mutation (create / update / delete) persists after page refresh
Logging Standard
Markdown bullet list per step → Observation, Result (pass/fail), Screenshot link
Capture full-page screenshot on failure
Use timestamp prefix YYYY-MM-DD_HH-MM in saved screenshot filenames
