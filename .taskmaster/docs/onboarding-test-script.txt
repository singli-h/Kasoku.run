ONBOARDING FLOW - COMPREHENSIVE TEST SCRIPT
============================================

Goal: Test the new multi-step onboarding wizard for both athlete and coach roles

Pre-requisites:
- User must be signed in to Clerk
- User should not have completed onboarding yet
- Test on both localhost:3000 (dev) and production URL

TEST CASE 1: ATHLETE ONBOARDING FLOW
====================================

Step 1: Navigate to Onboarding
- Go to /onboarding
- Verify the welcome step loads with:
  - Welcome message and icon
  - "Get Started" button
  - Progress bar showing 0% completion
- Take screenshot

Step 2: Welcome Step
- Click "Get Started" button
- Verify transition to role selection step
- Progress bar should show ~16% completion
- Take screenshot

Step 3: Role Selection
- Verify two role cards are displayed:
  - Athlete card with trophy icon
  - Coach card with users icon
- Click on "Athlete" card
- Verify card becomes selected (highlighted)
- Click "Continue" button
- Verify transition to athlete details step
- Progress bar should show ~33% completion
- Take screenshot

Step 4: Athlete Details
- Fill in athlete-specific fields:
  - Height: "175" (cm)
  - Weight: "70" (kg)
  - Training Goals: "Improve 5K time to under 20 minutes"
  - Experience: "2 years recreational running"
  - Select events: Check "5000m" and "10000m"
- Verify all fields accept input correctly
- Click "Continue" button
- Verify transition to subscription step
- Progress bar should show ~50% completion
- Take screenshot

Step 5: Subscription Selection
- Verify two subscription options are displayed:
  - Free plan with basic features
  - Paid plan with premium features
- Select "Free Plan"
- Click "Continue" button
- Verify transition to dashboard tour step
- Progress bar should show ~66% completion
- Take screenshot

Step 6: Dashboard Tour
- Verify feature overview cards are displayed:
  - Performance Analytics
  - Training Calendar
  - Workout Library
  - Progress Tracking
- Click "Continue" button
- Verify transition to completion step
- Progress bar should show ~83% completion
- Take screenshot

Step 7: Completion
- Verify completion message is displayed
- Verify "Complete Setup" button is present
- Click "Complete Setup" button
- Verify loading state shows "Setting up your account..."
- Wait for completion (may take a few seconds)
- Verify redirect to /dashboard
- Take screenshot of dashboard

Step 8: Verify Database Records
- Check that user record was created/updated with:
  - role: "athlete"
  - onboarding_completed: true
  - subscription_status: "free"
- Check that athlete record was created with:
  - height: 175
  - weight: 70
  - training_goals: "Improve 5K time to under 20 minutes"
  - experience: "2 years recreational running"
  - events: ["5000m", "10000m"]

EXPECTED RESULTS
===============

Success Criteria:
✅ All 7 steps of onboarding complete without errors
✅ Proper form validation on all fields
✅ Smooth transitions between steps with animations
✅ Progress bar updates correctly
✅ Database records created correctly for both roles
✅ Redirect to dashboard after completion
✅ Mobile/tablet responsive design works
✅ Back navigation works correctly
✅ Loading states display properly

Failure Indicators:
❌ JavaScript errors in console
❌ Steps don't advance or get stuck
❌ Form validation doesn't work
❌ Database records not created
❌ No redirect after completion
❌ Layout breaks on different screen sizes
❌ Missing or broken animations
❌ Progress bar doesn't update

DEBUGGING TIPS
==============

If tests fail:
1. Check browser console for JavaScript errors
2. Check network tab for failed API calls
3. Verify Clerk authentication is working
4. Check Supabase connection and RLS policies
5. Verify environment variables are set correctly
6. Check server logs for backend errors

API Endpoints to Monitor:
- POST /api/users/onboard
- Supabase users table operations
- Supabase athletes/coaches table operations
