# Workout MVP PRD – Athlete Workout Flow (Main + History)

## 1. Vision & Scope
- **Vision**: Deliver a clean, reliable athlete workout experience that prioritizes ongoing/today sessions with simple superset handling and robust status/date logic.
- **In-Scope**: Main Workout page (/workout), History page (/workout/history), status enum migration, date filtering, improved session discovery, sample data seeding (dev), comprehensive error handling, performance optimization, Jest test coverage.
- **Out-of-Scope**: Video storage integration, offline/PWA, AI picker, advanced analytics.

## 2. Personas & User Stories
- **Athlete**: As an athlete, I want to quickly start or continue today's workout, so I can train without confusion.
- **Athlete**: As an athlete, I want to review past workouts, so I can track progress.
- **Coach (secondary)**: As a coach, I want athletes to see assigned sessions for today first, so adherence improves.

## 3. Features & Dependency Chain

### A. Status & Data Foundations (DB, Types)
- Create Supabase enum `session_status = ('assigned','ongoing','completed','cancelled')`
- Migrate `exercise_training_sessions.status` → `session_status`
- Indexes: `date_time`; `(athlete_id, status)`; `(athlete_id, date_time, status)`
- Update types in `apps/web/types/database.ts` and related usages

### B. Session Discovery Logic (Server Actions)
- Implement `getTodayAndOngoingSessionsAction(athleteId?)` → ongoing first, then today assigned
- Implement `getPastSessionsAction(athleteId?, paging)`
- Update `startTrainingSessionAction` to set status assigned (then transition to ongoing on start)
- Add `updateTrainingSessionStatusAction(sessionId, status)`

### C. Workout UI/UX
- Main `/workout`: show ongoing + today; CTA to continue/start; clear empty state
- Keep superset visual grouping; no sequential enforcement; add "Complete Superset" helper
- Improve auto-save error toasts and retry; session timer stays

### D. History Page
- `/workout/history`: list past completed sessions (paginated), filters by date range optional (MVP: simple pagination)
- Link from dashboard and workout page

### E. Sample Data (Dev)
- Seed: 1 ongoing, 1 today assigned, 3 past completed (with supersets)
- Use MCP to insert rows in dev Supabase project

### F. Error Handling & Performance
- Standard ActionState errors; user-friendly empty/failed states; retry for loads/saves
- LRU caching for session data; TanStack Query optimization
- Error boundaries with consistent UI patterns

## 4. Success Metrics & Release Criteria & Timeline
- **Metrics**: % sessions started from today card; error rate < 1%; load time < 1.5s (server)

## 8. Error Handling & Edge Cases

### 8.1 Potential Issues & Mitigations

#### Database Issues
- **Issue**: Enum migration fails
- **Mitigation**: Fallback mapping in code, rollback plan
- **Code**: `const status = session.status as SessionStatus || 'unknown'`

#### Network Issues
- **Issue**: Auto-save fails
- **Mitigation**: Retry with exponential backoff, offline queue
- **Code**: Use existing retry patterns from useSupabaseQuery

#### State Issues
- **Issue**: Session status mismatch
- **Mitigation**: Validation on load, status reconciliation
- **Code**: `validateSessionStatus(session)`

#### Timezone Issues
- **Issue**: "Today" filter incorrect
- **Mitigation**: Server UTC with user timezone consideration
- **Code**: `isToday(date, userTimezone)`

### 8.2 Error Boundary Implementation
```typescript

```
