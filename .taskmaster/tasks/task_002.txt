# Task 2: Implement Standardized Data Fetching Pattern

## Description
Create a standardized data fetching pattern using React Query and the singleton supabase client, **and migrate the entire code-base to use it**. Fix all current mistakes where Clerk UUIDs are used directly in SQL and ensure every server/client interaction obeys the 2025 Supabase ⇄ Clerk integration rules.

## Current Repository Analysis

### ✅ Existing Infrastructure (Good Foundation)
- **Singleton Supabase Client**: Already implemented in `apps/web/lib/supabase-server.ts` with proper 2025 Clerk integration
- **User Cache System**: `apps/web/lib/user-cache.ts` provides LRU caching for Clerk ID → DB user ID mapping (15min TTL, 100 entries)
- **React Query Setup**: `apps/web/components/utilities/providers/query-provider.tsx` already configured
- **ESLint Rules**: Custom rules enforce singleton client usage and prevent direct `createClient` calls
- **ActionState Pattern**: Consistent error handling across server actions

### ❌ Current Issues Found (Need Fixing)
- **Widespread Clerk UUID Usage**: 50+ instances of `.eq('clerk_id', userId)` across actions
- **Mixed Data Fetching**: Some components use direct Supabase calls instead of server actions
- **Inconsistent Patterns**: Some actions use `getDbUserId()` while others manually query users table

### 🔍 Specific Problem Areas Identified
- `training/training-plan-actions.ts`: 15+ instances of `.eq('clerk_id', userId)`
- `training/training-session-actions.ts`: 3 instances of `.eq('clerk_id', userId)`
- `training/athlete-actions.ts`: 5+ instances of `.eq('clerk_id', userId)`
- `auth/user-actions.ts`: Already mostly correct but needs consistency
- Client components: `ExistingPlansTab`, `SprintSessionPresets` using legacy patterns

### 🚨 Critical ESLint Rule Gap Found
**ISSUE**: The existing `no-recreate-supabase.js` ESLint rule has a **critical limitation**:
- ✅ **Works for server-side**: Prevents `createClient` usage in server actions
- ❌ **Skips client-side**: Explicitly allows `createClientSupabaseClient` in client components
- **Problem**: This allows the very anti-pattern we want to prevent - direct Supabase calls in components

**Current Rule Logic**:
```javascript
// Skip client-side files (those with "use client" directive)
const text = sourceCode.getText()
if (text.includes('"use client"') || text.includes("'use client'")) {
  return {} // ❌ RULE BYPASSED FOR CLIENT COMPONENTS
}
```

**SOLUTION NEEDED**: Enhance the ESLint rule to also restrict `createClientSupabaseClient` usage in client components, encouraging server action usage instead.

## Critical Issues & Bad Approaches Identified

### ❌ Hard-Coded Approach in Original Hook Design
```typescript
// ❌ BAD: Hard-coded server action call
const userResult = await getCurrentUserAction()
if (!userResult.isSuccess || !userResult.data) {
  throw new Error('User not authenticated')
}
return fetchFn(supabase, userResult.data.id) // Use DB user ID
```

**Problems**:
- **Hard-coded dependency**: The hook is tightly coupled to `getCurrentUserAction()`
- **No flexibility**: Can't use different user context methods
- **Performance issue**: Every query calls the same server action
- **Type safety**: No guarantee `userResult.data.id` exists

### ❌ Lazy Error Handling
```typescript
// ❌ BAD: Generic error message
throw new Error('User not authenticated')
```

**Problems**:
- **No error context**: Doesn't distinguish between auth failure vs network issues
- **No retry logic**: Fails immediately on first auth issue
- **Poor UX**: Generic error doesn't help users understand the problem

### ❌ Inconsistent Pattern Application
The original approach suggests using `getDbUserId()` but then shows examples that don't follow this pattern consistently.

## Refined Implementation Approach

### Phase 1: Fix Server Actions (Priority 1)
```typescript
// CORRECTED PATTERN for all server actions
import { getDbUserId } from "@/lib/user-cache"
import supabase from "@/lib/supabase-server"
import { auth } from "@clerk/nextjs/server"

export async function someAction(): Promise<ActionState<T>> {
  const { userId } = await auth()
  if (!userId) return { isSuccess: false, message: "Not authenticated" }
  
  const dbUserId = await getDbUserId(userId) // Use cached lookup
  
  const { data, error } = await supabase
    .from('some_table')
    .select('*')
    .eq('user_id', dbUserId) // Use INT, not UUID
}
```

### Phase 2: Create Flexible Client Hook (Priority 2)
```typescript
// hooks/useSupabaseQuery.ts - IMPROVED VERSION
import { useQuery, QueryKey, UseQueryOptions } from '@tanstack/react-query'
import { createClientSupabaseClient } from '@/lib/supabase-client'
import { useAuth } from '@clerk/nextjs'
import { getCurrentUserAction } from '@/actions/auth/user-actions'

// ✅ BETTER: Specific error types
export class SupabaseQueryError extends Error {
  constructor(
    message: string,
    public code: 'AUTH_FAILED' | 'NETWORK_ERROR' | 'DATABASE_ERROR',
    public originalError?: Error
  ) {
    super(message)
    this.name = 'SupabaseQueryError'
  }
}

export function useSupabaseQuery<T>(
  key: QueryKey,
  fetchFn: (supabase: SupabaseClient, dbUserId: number) => Promise<T>,
  options?: UseQueryOptions<T> & {
    userContextFn?: () => Promise<{ id: number }>
  }
) {
  const { getToken } = useAuth()
  
  return useQuery<T>({
    queryKey: key,
    queryFn: async () => {
      const supabase = createClientSupabaseClient(getToken)
      
      // ✅ BETTER: Configurable user context
      const userContextFn = options?.userContextFn || getCurrentUserAction
      const userResult = await userContextFn()
      
      if (!userResult.isSuccess || !userResult.data?.id) {
        throw new SupabaseQueryError(
          'User not authenticated or missing ID',
          'AUTH_FAILED'
        )
      }
      
      return fetchFn(supabase, userResult.data.id)
    },
    // ✅ BETTER: Smart retry logic
    retry: (failureCount, error) => {
      // Don't retry auth errors
      if (error instanceof SupabaseQueryError && error.code === 'AUTH_FAILED') {
        return false
      }
      return failureCount < 3
    },
    retryDelay: (attemptIndex) => Math.min(1000 * 2 ** attemptIndex, 30000),
    ...options,
  })
}
```

### Phase 3: Enhanced ESLint Rule (Priority 1)
```javascript
// ✅ BETTER: Enhanced ESLint rule for client-side restrictions
// apps/web/eslint-rules/no-recreate-supabase.js
module.exports = {
  meta: {
    type: 'problem',
    docs: {
      description: 'Prevent direct usage of createClient and createClientSupabaseClient',
      category: 'Best Practices',
      recommended: true,
    },
    fixable: null,
    schema: [],
    messages: {
      noRecreateSupabase: 'Use the singleton Supabase client from "@/lib/supabase-server" instead of creating a new client directly.',
      noClientSupabase: 'Use server actions instead of direct Supabase client calls in components.',
    },
  },

  create(context) {
    const filename = context.getFilename()
    const sourceCode = context.getSourceCode()
    
    // Check for client-side files
    const text = sourceCode.getText()
    const isClientFile = text.includes('"use client"') || text.includes("'use client'")
    
    // Skip the singleton helper files
    if (filename.endsWith('supabase-server.ts') || filename.endsWith('supabase-client.ts')) {
      return {}
    }
    
    // Skip test files and node_modules
    if (filename.includes('.test.') || filename.includes('.spec.') || 
        filename.includes('__tests__') || filename.includes('node_modules') || 
        filename.includes('.next')) {
      return {}
    }

    return {
      // Check for createClient imports (server-side)
      ImportDeclaration(node) {
        if (node.source.value === '@supabase/supabase-js') {
          const hasCreateClient = node.specifiers.some(spec => {
            return (
              (spec.type === 'ImportSpecifier' && spec.imported.name === 'createClient') ||
              (spec.type === 'ImportNamespaceSpecifier') ||
              (spec.type === 'ImportDefaultSpecifier')
            )
          })
          
          if (hasCreateClient && !isClientFile) {
            context.report({
              node,
              messageId: 'noRecreateSupabase',
            })
          }
        }
      },
      
      // Check for createClientSupabaseClient usage (client-side)
      CallExpression(node) {
        if (isClientFile && node.callee.name === 'createClientSupabaseClient') {
          context.report({
            node,
            messageId: 'noClientSupabase',
          })
        }
        
        // Check for direct createClient calls (server-side)
        if (!isClientFile && node.callee.name === 'createClient') {
          context.report({
            node,
            messageId: 'noRecreateSupabase',
          })
        }
      },
    }
  },
}
```
