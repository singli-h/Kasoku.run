5. Critical Pitfalls to Avoid
❌ DON'T:
Use h-screen or h-full in nested components
Why: Conflicts with parent sizing, creates double scroll
Exception: Only use in top-level page wrapper if needed
Nest overflow-* containers
Why: Child can't scroll if parent blocks it
Rule: Only ONE overflow container per scroll direction
Use flex-1 with explicit heights
Why: Flex-1 means "take remaining space" but h-screen/h-full overrides it
Rule: Choose one: flex-1 OR explicit height, never both
Add overflow-x-hidden to parents of scrollable content
Why: Blocks horizontal scroll from children
Rule: Only add overflow-x-auto to the immediate parent of wide content
Use w-full on tables meant to scroll
Why: Forces table to fit container, preventing overflow
Rule: Let tables be natural width, add min-w-[...] to trigger scroll
Add min-w-0 everywhere "just in case"
Why: Can prevent proper layout, use only when flex children won't shrink
Rule: Only use when debugging flex wrap issues
✅ DO:
Use simple spacing utilities (space-y-4, gap-4)
Why: Clean, predictable vertical/horizontal rhythm
Put overflow-x-auto on immediate parent of scrollable content
Why: Direct parent-child relationship for scroll behavior
Use semantic HTML (<table>, not <div role="table">)
Why: Better accessibility, native browser behavior
Position overlays with fixed or absolute
Why: Don't interfere with document flow
Test on actual mobile devices, not just DevTools
Why: Touch scrolling behaves differently than mouse scroll
Keep nesting shallow (max 4-5 levels)
Why: Easier to debug, better performance
6. Lessons Learned from Failures
Failure 1: "Just add overflow-x-auto"
What happened: Added overflow-x-auto to table but parent had overflow-x-hidden Lesson: Always check parent chain for conflicting overflow
Failure 2: "Add width constraints everywhere"
What happened: Added w-full max-w-full min-w-0 throughout chain Lesson: Too many constraints fight each other, keep it simple
Failure 3: "Copy Shadcn/MUI pattern"
What happened: Pattern works standalone, but conflicts in nested layout Lesson: Patterns must be adapted to context, not copy-pasted
Failure 4: "Fix global layout"
What happened: Added min-w-0 to ProtectedLayout, broke all pages Lesson: Never change global layout for one page's issues
Failure 5: "Nested flex containers"
What happened: Multiple flex flex-col with flex-1 created sizing chaos Lesson: Avoid flex nesting, use simple block layout
