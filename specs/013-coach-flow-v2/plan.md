# Implementation Plan: Coach Flow V2

**Branch**: `013-coach-flow-v2` | **Date**: 2026-03-11 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/013-coach-flow-v2/spec.md`

## Summary

Enable the first beta coach to migrate from TrainHeroic by adding exercise-level subgroup filtering, rich workspace session cards, template exercise insertion, a dashboard mini calendar, and AI text-to-blocks parsing. The implementation spans DB schema (1 new column + RLS), server actions (3 new, 3 modified), shared utilities (2 new), and 5 UI feature areas touching ~25 files.

## Technical Context

**Language/Version**: TypeScript 5.x, React 19, Next.js 16
**Primary Dependencies**: Supabase (PostgreSQL + RLS), TanStack Query, Tailwind CSS, date-fns, shadcn/ui, Anthropic Claude API (structured output)
**Storage**: Supabase PostgreSQL (hosted), RLS-enforced
**Testing**: Vitest (unit), agent-browser (E2E)
**Target Platform**: Web (mobile-first responsive)
**Project Type**: Turborepo monorepo — `apps/web/` (Next.js)
**Performance Goals**: Workspace loads <2s, subgroup filter preview instant (client-side), calendar week switch <500ms
**Constraints**: RLS must enforce subgroup filtering (not just UI). Monday week start convention. 3-char max tag display.
**Scale/Scope**: Single coach beta, ~50 athletes across 6-10 subgroups

## Risk Mitigation

| Risk | Impact | Mitigation |
|---|---|---|
| RLS policy complexity causes query slowdown | Athlete workout load time | GIN index on target_event_groups; benchmark with 50+ exercises |
| Type regeneration strips custom aliases (again) | Build breaks | Documented in quickstart.md — always restore after `generate_typescript_types` |
| workout_log_exercises copy step misses filtering | Athletes see wrong exercises on session start | Filter at both copy step AND RLS level (defense in depth) |
| AI text parser produces bad structured output | Coach frustration | Preview + edit step before insertion; flagging unparseable lines |
| Templates page rewrite breaks existing template data | Lost templates | Only changing UI layer — DB actions (getTemplatesAction, deleteTemplateAction) already work |
